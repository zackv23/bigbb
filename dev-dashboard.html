Auto-Save Journal
Every filesystem change automatically stages and commits locally
⌚
Git WIP Watcher
Python file watcher in background
Watches files in backend/app and backend/tests. Whenever a save is completed, it commits immediately with: WIP: file_name @HH:MM:SS. Allows safe code iteration.
python3 .dev-watcher.py &
📒
Session Journal Log
.dev-logs/session.jsonl
Each file-write event records a JSON structured line in the local session journal storing the file path, epoch timestamp, git SHA, and message payload.
tail -f .dev-logs/session.jsonl
📦
Database & Code Backups
Hourly automated snaps saved directly to user Desktop folder
🗄️
Hourly SQL Dumps
pg_dump → .sql.gz
Backup targets the local PostgreSQL database, archives schemas, settings, and tables, zips it, and places it in ~/Desktop/BigBB TERMINAL/.
🗜️
Hourly Code Zip
Full zip snapshot of bigbb/
Zips the entire workspace directory, ignoring massive binary caches (node_modules/, .venv/, .git/) to create extremely light zip files.
🔁
Backup Cron Daemon
Background loops
`dev.sh` runs a background task loop that calls `backup_db` and `backup_code` every 60 minutes automatically while you are working.
sleep 3600 && backup_both
↩
Local Recovery & Restorations
Drops current DB and checks out code blocks to restore previous states
💿
Interactive Database Restore
Overwrites current postgres data
Running database restore opens an interactive list of backups in your terminal. Selecting one drops bigbb database, recreates it, and loads SQL dump.
bash dev.sh restore-db
📂
Code Restoration (Git rollbacks)
Resets local staging files
Replays the auto-save WIP commits. You can easily pick a git hash to checkout a single file or restore the entire repository to that exact timestamp.
bash dev.sh restore-code

Command-Line Reference (dev.sh)
Standard commands to run inside the terminal workspace
✓ dev.sh is already written and placed at ~/Desktop/bigbb/dev.sh.
## SERVICE MANAGEMENT bash dev.sh # start all services + watcher + auto-backup bash dev.sh stop # kill all background processes cleanly bash dev.sh logs # pretty-print live log stream ## TESTING PIPELINES bash dev.sh test # run full pytest suite bash dev.sh test-watch # pytest re-runs on every .py save ## SNAPSHOT UTILS bash dev.sh backup-db # pg_dump → Desktop/BigBB TERMINAL/ bash dev.sh backup-code # zip snapshot → Desktop/BigBB TERMINAL/ bash dev.sh restore-db # interactive: restore psql from dump
